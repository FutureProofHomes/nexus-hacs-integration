# Security Policy

## Supported Versions

We encourage reporting vulnerabilities for the latest release. As this is a community-maintained
integration, older versions may not receive patches.

## Reporting a Vulnerability

Please report security vulnerabilities privately via email or by opening a private GitHub issue.
Do **not** open a public issue for security-sensitive bugs.

Once reported, we will:

1. Acknowledge receipt within 48 hours.
2. Assess the severity and assign a CVE if applicable.
3. Publish a patch in a timely manner.
4. Credit the reporter (unless anonymity is requested).
